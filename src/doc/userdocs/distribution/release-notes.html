Fedora 2.1.1 Release Notes

April 7, 2006

Table of Contents

  1. Notable Bug Fixes
  2. Known Issues
  3. Installation

Fedora 2.1.1 is a maintenance release that addresses a number of bugs and performance issues in the 2.1 release. A detailed analysis of the performance of Fedora API operations is available [here].

In Fedora 2.1.1, the LowLevelStorage component has been refactored as a module. If you are upgrading from a previous version of Fedora, you should note that the server parameters related to LowLevelStorage (namely, object_store_base, datastream_store_base, path_registry, path_algorithm, file_system, and backslash_is_escape are now found in the LowLevelStorage module configuration section of the Fedora Configuration.

  1. Notable Bug Fixes

    Refer to Fedora's Bugzilla for full descriptions of the resolved bugs.

    • Bug #145: FedoraClient's HTTP header date parsing fails in non-english locales
    • Bug #146: ResourceIndex Rebuilder error with relative path for triplestore
    • Bug #147: SQL database left in corrupted state after ingest of object with label exceeding 255 characters
    • Bug #148: Admin client bDef/bMech builders fail when specifying system generated PID
    • Bug #149: adding sample policy results in NullPointerException
    • Bug #151: surrogate policies generate errors in console log
    • Bug #152: modifyDatastreamByValue delay when resourceIndex on
    • Bug #154: Resource Index datastore parameter, poolMaxGrowth, doesn't behave as documented
    • Bug #156: Modifying a disseminator's bmech breaks disseminations for other objects
    • Bug #157: Deleting digital object leaves behind triples in Resource Index
    • Bug #158: purgeObject and purgeDisseminator take unreasonably long in large repositories
  2. Known Issues
    • An early adopter of Fedora 2.1 reported anomalous 401 status codes returned after a period of normal server use. This problem is difficult to replicate, and the cause is unknown. A workaround to this problem is to instruct Tomcat to use a MemoryRealm instead of a JAASRealm. See Bug 144 for detailed instructions on applying this workaround.
    • Ingest validation error when namespace for in-line XML datastreams is in root element. Reported as Bug 153.
    • fedora-reload-policies.sh gives usage error if run with no arguments. Reported as Bug 160.
  3. Installation

    For installations and upgrades of Fedora, please see the Installation Guide and the Upgrade and Migrate Guide.

Fedora Search Interface

Introduction

The Search Interface provides a mechanism for searching and browsing the Fedora repository. Upon ingestion, metadata from the Fedora System Metadata section and the Dublin Core Metadata section of the object are indexed in a relational database . System Metadata contains metadata maintained by the Fedora repository that consists of items such as object PID, object label, object creation date, etc. Dublin Core Metadata is an optional Implementor-Defined XML Metadata datastream in the object that contains Dublin Core elements describing the object.. If no Dublin Core record is provided, the system will attempt to construct a minimal Dublin Core record based on information in the System Metadata consisting of dc:title corresponding to the object's label and dc:identifier corresponding to the object's persistent identifier (PID). This bare minimum Dublin Core record is required for the OAI Provider service to disseminate metadata about each object in the repository.

The search tool interface provides both simple and advanced searching. Simple search enables searching across all metadata files. Advanced Search enables fielded searching across any combination of metadata elements using Boolean operators. Wildcard features are supported in both search types.
